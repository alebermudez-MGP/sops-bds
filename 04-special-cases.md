# ⚠️ 04 — Special Cases

Reference this page when a file involves Concurrent accounts, Delinquency, Collection Review, Portfolio switches, Re-enrollment, or Bank Account Updates.

---

## Section 1 — Concurrent Accounts

<details>
<summary>➕ When a Concurrent should be created</summary>
<br>
A Concurrent is created when:
<ul>
<li>The same business name and same owner are involved,</li>
<li>But the creditor is different.</li>
</ul>
If the creditor is the <strong>same</strong>, confirm with Sales whether this is a second position to avoid creating a duplicate account for an existing lender.
<br><br>
In many cases, merchants request to add a new creditor to the program after they are already enrolled and have made one or more payments. Before creating a Concurrent, review all of the following items carefully.
</details>

<details>
<summary>✅ 1. Enrolled Account</summary>
<br>
A Concurrent should only be created if the original account is already enrolled.
</details>

<details>
<summary>📅 2. 1st Payment Status</summary>
<br>
<ul>
<li><strong>If the 1st payment is in Open status:</strong>
<ul>
<li>Attempt to avoid creating a Concurrent.</li>
<li>Modify the current account instead and send a new contract with the same FPD.</li>
<li>Remember to check the cutoff time.</li>
</ul>
</li>
<li><strong>If the 1st payment is in Processed or Cleared status:</strong>
<ul><li>You must proceed with a Concurrent.</li></ul>
</li>
<li><strong>If a CC payment is made before the scheduled 1st payment:</strong>
<ul><li>You may proceed with a Concurrent only if the opportunity notes confirm a Secured CC Payment, or the Card Payment Authorization Form is signed and uploaded.</li></ul>
</li>
</ul>
</details>

<details>
<summary>↩️ 3. Recent Returns</summary>
<br>
<ul>
<li>If there are recent returns, the file may need to be handled as a delinquency instead of a Concurrent.</li>
<li>Review the file and notes thoroughly to confirm whether the return has been rescheduled.</li>
<li>The merchant must address the return(s) or the account must show a confirmed reschedule before you proceed.</li>
<li>If the above is met, you may continue with the Concurrent.</li>
</ul>
</details>

<details>
<summary>🕐 4. 1st Payment Date in Open Status (Same-Day Batch)</summary>
<br>
If the batch date is the same day and we are before cutoff:
<ul>
<li>Inform the Sales Agent that a Concurrent cannot be created yet.</li>
<li>Provide the two available options:
<ol>
<li>Send a new contract with updated creditor information while keeping the same FPD.</li>
<li>Wait until after cutoff and then proceed with the Concurrent.</li>
</ol>
</li>
</ul>
</details>

<details>
<summary>⚠️ 5. Recent Returns Not Cleared or Rescheduled</summary>
<br>
<ul>
<li>If the recent returns remain unresolved or not rescheduled, notify the Sales Agent that the file must be handled by the CS Team.</li>
<li>Once the account returns to ACTIVE status, you may proceed with the Concurrent request.</li>
</ul>
</details>

<details>
<summary>📋 Concurrent Requirements</summary>
<br>
<ul>
<li>Any concurrent file must be handled under the same portfolio as the original file.<br>
Example: If a merchant has Business #1 with one lender under BDS, and later enrolls Business #1 with a different lender as a concurrent file, the concurrent file must also be completed under BDS.</li>
<li>When we receive a request to create a concurrent account, we will assign the <strong>same client service agent</strong> who is already assigned to the original account, to ensure continuity and consistency.</li>
<li>Concurrents do not apply to different debts — a concurrent applies only when the debt type is the same within the same portfolio. If it is a different debt, even if it belongs to the same client, it should not be considered concurrent.</li>
</ul>
<br>
📩 <strong>QA Reminder:</strong> To avoid errors, carefully verify the debt type and portfolio before creating or assigning a file as concurrent. Double-check this information to ensure accuracy in our processes.
</details>

## Section 2 — Delinquency

<details>
<summary>📌 Delinquency Status categories</summary>
<br>
<ul>
<li>Information Collection <em>(check with management before proceeding with Delinquent)</em></li>
<li>Program Hold / Paused <em>(check with management before proceeding with Delinquent)</em></li>
<li>Cancellation Requested <em>(check with management before proceeding with Delinquent)</em></li>
<li>Cancellation Approved</li>
<li>Cancellation Processed</li>
<li>Need to Process Refund</li>
<li>Refund Processed</li>
<li>Client Defaulted</li>
<li>Cancellation Fee Paid</li>
<li>Client Canceled</li>
<li>Canceled</li>
<li>Collections Review</li>
</ul>
</details>

<details>
<summary>🚫 No double delinquency rule</summary>
<br>
We will <strong>no longer enroll any file that has been delinquent more than once</strong>, regardless of whether the delinquency occurred under the same business name or under a different business name with the same owner.
<br><br>
In the event that the same owner has a different business name, and one or more accounts are in <strong>"Collections Review"</strong> status or have recent returns that have not been cleared, we may proceed with the new account under the new business name. However, the merchant must submit a <strong>credit card (CC) payment</strong> for the required fees before the account can be enrolled in the program.
<br><br>
Add all relevant notes to the system for proper documentation: <em>"Merchant must make a payment for the Max CC Payment Amount ($X.XX) to be enrolled."</em> and inform the Sales Agent of this requirement.
</details>

<details>
<summary>📝 Underwriting Team Guidelines for delinquent accounts</summary>
<br>
The UW team may move forward with a request involving a delinquent account only after following these specific steps:
<ol>
<li><strong>Review for duplicates.</strong> If an account in delinquency status is found, review and take into account the notes on the file in delinquency status before making any decisions or requesting clarification from the Sales Team.</li>
<li><strong>Verify the cancellation fee.</strong> Look for any notes regarding cancellation fees and add up all amounts to determine the total cancellation fee. These notes can be under Management MGP or a manager.</li>
</ol>
</details>

## Section 3 — Collection Review

<details>
<summary>📂 Step 1 — Confirm file status</summary>
<br>
Verify the file status before proceeding with any action on a file under Collection Review.
</details>

<details>
<summary>✉️ Step 2 — Send email to Sales Agent</summary>
<br>
Send an email to the Sales Agent to confirm whether the merchant intends to continue with the file.
<br><br>
<strong>If the Merchant Decides to Continue With the File:</strong>
<ol>
<li>The merchant must be brought back on track.</li>
<li>Underwriting (UW) will email Customer Service (CS) requesting assistance in helping the merchant get back on track and advising them to follow up with the Sales Agent.</li>
<li>Once the merchant is back on track, the status can be updated or transactions reactivated. This action may be completed by either UW or CS.</li>
</ol>
💡 CS email reference:<br>
BDS: <strong>clientsupport@businessdebtsolutions.com</strong>
<br><br>
<strong>If the Merchant Decides Not to Continue With the File:</strong>
<ol>
<li>Proceed with creating a new opportunity following the established underwriting process and using the proper pricing.</li>
<li>Once the program cost matches the calculator amount, add a 2-month add-on fee, which will increase the program cost.</li>
</ol>
⚠️ If the merchant has already paid a substantial amount of fees and Sales submits a new request, consult with your immediate Supervisor or Manager to determine if an exception can be made.
</details>

## Section 4 — Portfolio Switches

<details>
<summary>🔄 Correct process for merchant portfolio switches</summary>
<br>
To switch from one portfolio to another (BDS or DAG), follow these steps carefully to ensure clean documentation and accurate records.
<br><br>
<strong>Close the original portfolio file:</strong>
<ol>
<li>Update the Underwriting Status to <strong>Withdrawn</strong>.</li>
<li>If a contract was already sent, void it immediately.</li>
<li>Add an internal note: <em>"Client does not want to continue with this plan."</em></li>
</ol>
<strong>Start work under the new assigned portfolio:</strong>
<ol start="4">
<li>Begin processing the file under the new portfolio as recommended by management.</li>
<li>Continue following the standard underwriting (UW) steps as usual.</li>
</ol>
💡 Reminder: the Underwriting stage and Withdrawal status will continue to be used when we switch an opportunity to the other portfolio.
</details>

## Section 5 — Re-enrollment

<details>
<summary>🔁 Re-enrolling a merchant after a withdrawn or canceled file</summary>
<br>
When a merchant who previously did not continue (see Collection Review above) decides to come back, treat it as a new opportunity rather than reactivating the old one:
<ul>
<li>Create a new opportunity following the established underwriting process and using the proper pricing.</li>
<li>Once the program cost matches the calculator amount, add a <strong>2-month add-on fee</strong>, which will increase the program cost.</li>
</ul>
⚠️ If the merchant already paid a substantial amount of fees previously, consult your immediate Supervisor or Manager before proceeding — an exception may apply.
</details>

## Section 6 — Bank Account Updates

<details>
<summary>🏦 Required documentation for bank account updates</summary>
<br>
This policy applies to all bank account updates requested by clients — both <strong>existing enrolled clients</strong> and those <strong>requesting a new contract</strong>.
<br><br>
To process the client's request, you must secure <strong>one</strong> of the following forms of documentation:
<ul>
<li>A valid <strong>Voided Check</strong>, or</li>
<li>An official <strong>Bank Letter</strong> AND the client's <strong>Most Recent Bank Statement</strong>.</li>
</ul>
⚠️ <strong>Starter Checks and Direct Deposit Forms are no longer valid documents</strong> for this verification process.
<br><br>
Requests submitted without one of the approved options will be <strong>delayed</strong> until the correct documentation is provided.
</details>

<details>
<summary>🆕 Exception — brand new accounts with limited documentation</summary>
<br>
For the specific scenario of a <strong>brand new account</strong> where the merchant cannot provide a valid Voided Check or recent statements, you must obtain <strong>both</strong> of the following documents:
<ul>
<li>A <strong>Starter Check</strong></li>
<li>An official <strong>Bank Letter</strong></li>
</ul>
This stricter requirement for new accounts is necessary to ensure the provided information is valid before continuing with the process.
</details>
